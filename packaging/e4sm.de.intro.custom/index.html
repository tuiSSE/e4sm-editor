Welcome to the E4SM Editor

Getting Started

Creating a E4SM Model:

first off you need to create a modelling project, go through the toolbar like so  - File > New > Modelling Project

now choose a name - Project name: <ExampleProject>

then press finish

 


in order to create a E4SM model you need to right-click on your new modelling project and make your way through like this - New > Other... > E4SM Model Creation Wizard > E4SM Model

now choose a name ending in .e4sm - File name: <ExampleModel.e4sm>

then press finish

 


extending the model requires you to double-left-click a .e4sm  model-file, you should be able to see a kind of collapsed "path"

click on the small arrow to make the Modell component appear, right-clicking this "Modell" gives you the option to start creating parts via the New Child functionality

at this point you can freely build whatever model you want to

 


Using other Representations:

the E4SM Editor provides the option to build a package via drag-and-drop
if you want to use this functionality you will need to have already created a package via the New Child functionality in a model

all you then need to do is right-click the package in the left-side model explorer and follow this - New Representation > [DTD] 

choose a name and press ok, the view should automatically open itself along with the drag-and-drop palette on the right side

 

other useful representations can be created for the model-component, such as a Class Diagram, Documentation Table and Person Management Table

 


 

Undoing components:

undoing the creation or deletion of your latest component can be achieved via right-clicking the Modell component and choosing the "Undo <...>" option

the "Redo <...>" option cancels out the "Undo <...>" option, if there ever arises a need to undo the latest undo

 


Using text-format to create a model:

with this editor it's also possible to create a model only using text

first you will need to manually create a new file with .e4smcode as its ending

in this file one can now simply write like so - 

model binaryModel{
package p1 {
sensor s1 {
out s1_out
},
binaryClassificationComponent b1 {
in b1_in,
out b2_out
numberOfServers 3 confusionMatrixes { confusion matrix cm1 {TP : 200, FN : 2, FP: 5, TN: 8}
}
},
actuator a1 {
in a1_in
}
connector c1 "s1.s1_out" -> "b1.b1_in",
connector c2 "b1.b2_out" -> "a1.a1_in"
}
}
